CRUDScope – HTML MediaPanel

lo que se busca es estruturar una pagina donde se puedan cumplir las tareas principales de un crud tales como agregar,eliminar y actualizar.

ademas de eso practicar diferentes etiquetas que tiene html y ver cuales son sus funcionalidades,en esta ocacion no se esta buscando tener todo funcional, pero la idea es crear una especie de "esqueleto" para ver como podria quedar todo la pagina,obviamente cuidando la semantica y buscando tener buenas practicas para que todo sea lo mas profecional posible.

LISTAS DE ETIQUETAS USADAS

Estructura: html, head, body, title, base, meta.

uso: estas son las etiquetas principales y las que se debe cuidar de que tengan un orden correcto,como en el caso de html que es la etiqueta pricipal debe estar en la parte externa de todo el cofigo

Semánticas: header, nav, main, section, article, aside, footer, address.

uso: estas en parte como las anteriores se usan para que la pagina tenga un orden correcto.por ejemplo el header y el nav deben estar junto y nmo puede haber un footer dentro de un header

No semánticas: div.

uso:esta se usa para contener partes del codigo, por ejemplo si meto una parte del codigo en un div y le cambio el color todo lo que este en el div va a cambiar de color 

Texto: h1–h3, p, strong, em, u, mark.

uso:estas se usan para titulos y estilos de escritura.por ejemplo el mark le da un estilo diferente al texto

Tablas: table, tr, th, td.

uso:estas se usan para las tablas. por ejemplo si necesitamos hacer una tabla que contiene id,producto,precio,fecha de vencimiento

Formularios: form, label, input, textarea, button.

uso:estas se usan para la informacion que necesitamos del usuario.por ejemplo un formulario para pedirle los datos al usuario

Multimedia: img, audio, video.

uso:estas se utilizan para poder tener archivos multimedia en la pagina.por ejemplo para mostrar un video.

Diferencias entre etiquetas semánticas y no semánticas:

semanticas:Las etiquetas semánticas describen claramente el significado, propósito y tipo de contenido que contienen, tanto para las personas como para las máquinas (navegadores, motores de búsqueda y lectores de pantalla).

no semanticas:Las etiquetas no semánticas sirven como contenedores genéricos cuyo único propósito es agrupar contenido para aplicar estilos con CSS o manipularlo con JavaScript. No transmiten ningún significado sobre el tipo de contenido que albergan.


BASE:lo que hace es que si hay algun enlace que no tenga contenido va a mandar a la pagina que se encuentra dentro de base

ABBR:Su propósito es identificar y proporcionar una descripción completa para una abreviatura o un acrónimo dentro de un texto.

TIME:La etiqueta <time> sirve para dos cosas principales:

Marcar el Texto: Encierra un momento en el tiempo, como "20 de abril de 2025" o "18:00".

Formato de Máquina: Utiliza el atributo datetime para proporcionar la fecha y/u hora en un formato estandarizado que las máquinas pueden entender (ISO 8601).
